**Обобщённое программирование (Generics)** в Go позволяет создавать функции и типы, которые работают с любыми типами данных. Generics были введены в Go 1.18.

## Зачем нужны обобщения?

Generics позволяют писать универсальный код, избегая дублирования функций для разных типов и потери типобезопасности при использовании пустого интерфейса `interface{}`.

## Основные концепции

1. **Параметры типа (Type Parameters)**: Указываются в квадратных скобках `[]`.
2. **Ограничения (Constraints)**: `any`, `comparable` или интерфейсы.

### Пример функции

```go
func Min[T any](a, b T) T {
    // Note: для сравнения < нужно ограничение constraints.Ordered, а не any
    // Но для примера структур any показывает синтаксис
    return a
}
```

Правильный пример с ограничением:

```go
import "golang.org/x/exp/constraints"

func Min[T constraints.Ordered](a, b T) T {
    if a < b {
        return a
    }
    return b
}
```

### Обобщённые типы

```go
type Pair[T any] struct {
    First  T
    Second T
}
```

### Ограничения типов

Go предоставляет пакет `constraints` (в `golang.org/x/exp/constraints` или встроенные в новых версиях).

- `constraints.Ordered`: Числа и строки.
- `constraints.Integer`: Целые числа.

Пользовательские ограничения:

```go
type Addable interface {
    int | float64
}
```

## Преимущества

1. **Повторное использование кода**.
2. **Безопасность типов** (в отличие от `interface{}`).
3. **Производительность** (мономорфизация компилятором).

## Заключение

Generics делают Go более гибким, позволяя писать универсальные алгоритмы и структуры данных.
